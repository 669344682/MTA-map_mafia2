CollEditor2 - это прога для получения коллизии

DeleteZeroAreaFaces.ms - удаляет невидимые стены у коллизии объекта, переместить в скрипты 3д макса и запустить на выделенном объекте

FrameRes_Editor - это прога для получения положение объектов, нужно открывать sds файлы, и копировать часть которая начинается с Object,
потом надо вставить то, что скопировал в файл frame.txt, там при помощи нотепада++ заменить все запятые на точки и символ " на пустоту, потом выделить (Name: )
с пробелом и заменить на obj| так нужно чтобы с помощью проги SciTE которая устанавливается в LuaForWindows_v5.1.5-52, в файл записалось название объекта
и его положение, которое будет в файле frame_new.txt

josbalcaen_batch_exporter_importer - это скрипт для массового ипорта объектов в 3дс, гайд как установить в инете

KAM's_GTA_Scripts_Upd_by_GF_v0.3.4 - скрипты для экспорта объектов и коллизии

lua_mtl_obj - код для чтения файлов с расширение .obj, которые должны быть на диске С, в файле с расширение .mtl вместо hash текстуры записывается её реальное 
название, в программе SciTE открой его и запусти нажав run program

LuaForWindows_v5.1.5-52 - поставь голочку на установку SciTE, установится программа при помощи которой можно запускать луа скрипты

M2Toolkit_v2.001122019 - много возможностей, я в ней смотрю какие объекты мне экспортировать, если FrameRes_Editor не смог открыть файл, копирую положение вручную,
там справа есть надпись Matrix, нужно открыть её и там будет строка position, копируй её, также не забудь про ротацию, а потом замени запятые на точки и поставь
вместо y: z: тоже запятые, x: удали, а также в настройках в библиотеке указать 3 файла с расширеним .mtl, которые находятся в MafiaII\edit\materials

Model_Viewer - для экспорта объектов, экспортируй на диск С, так как этот путь указан в скрипте lua_mtl_obj 

pos_write - запускай скрипт через SciTE, он запишит имя и положение объекта в файл frame_new.txt

tt - запускай скрипт через SciTE, он найдет похожие объекты и закомментирует их записав в файл frame_new_copy.txt, потом это надо будет вставить в таблицу object
в скрипте lua_mtl_obj, чтобы не было похожих объектов
